================================================================================
DATABASE SCHEMA
================================================================================

TABLE: boq_fixed_structures (Template Definitions)
  - id (UUID PK)
  - company_id (FK) - Multi-tenancy
  - name (VARCHAR) - "BOQ-085 Residential Maisonette"
  - description (TEXT)
  - structure_data (JSONB) - Section/subsection definitions
  - is_active (BOOLEAN)
  - created_at, updated_at (TIMESTAMPTZ)
  - INDEX: company_id

TABLE: boq_fixed_items_v2 (Hierarchical Items)
  - id (UUID PK)
  - company_id (FK)
  - structure_id (FK)
  - section_id (TEXT) - "SECTION_A"
  - subsection_id (TEXT) - "MATERIALS"
  - item_number (VARCHAR) - "1", "2", "A", "B"
  - description (TEXT)
  - unit (TEXT)
  - default_qty (NUMERIC)
  - default_rate (NUMERIC)
  - sort_order (INTEGER)
  - created_at, updated_at (TIMESTAMPTZ)
  - INDEXES: company_id, structure_id, (section_id, subsection_id)

TABLE: boq_fixed_items_migration_log (Audit)
  - id (UUID PK)
  - company_id (FK)
  - old_item_count, new_item_count (INTEGER)
  - structure_id (FK)
  - status (VARCHAR) - "pending", "completed", "failed"
  - error_message (TEXT)
  - migrated_at (TIMESTAMPTZ)
  - created_at (TIMESTAMPTZ)

SERVICE API
================================================================================

STRUCTURE MANAGEMENT:
  - createStructure(companyId, name, description, structureData)
  - getStructures(companyId)
  - getStructure(structureId)
  - updateStructure(structureId, updates)

ITEM MANAGEMENT:
  - insertItems(items[])
  - getStructureItems(structureId)
  - updateItem(itemId, updates)
  - deleteItem(itemId)
  - deleteSubsectionItems(structureId, sectionId, subsectionId)

HIERARCHICAL DATA:
  - getHierarchicalData(structureId) → Returns complete tree with calculations

EXPORT:
  - exportForPDF(structureId) → Returns formatted data for PDF generator

MIGRATION:
  - migrateFromLegacy(companyId, legacyItems) → Import from old system

HOW TO USE
================================================================================

1. APPLY MIGRATION
   - Run: migrations/20250527_hierarchical_boq_schema.sql
   - Creates all new tables with proper indexes and RLS

2. CREATE BOQ STRUCTURE (via code or UI)
   Via Service:
   ```
   const struct = await hierarchicalBOQService.createStructure(
     companyId,
     'BOQ-085 Residential Maisonette',
     'Description',
     { sections: [...] }
   );
   ```

3. IMPORT ITEMS
   Via UI (/fixed-boq-hierarchical):
   - Click "Import BOQ"
   - Paste formatted BOQ text
   - System auto-detects sections, subsections, items
   - Click "Import"

   Or via Code:
   ```
   await hierarchicalBOQService.insertItems(items[]);
   ```

4. VIEW & EDIT
   - Navigate to /fixed-boq-hierarchical
   - Select structure from dropdown
   - Click sections to expand/collapse
   - Click edit icon to modify items
   - Totals update in real-time

5. EXPORT PDF
   - Click "Export PDF" button
   - Browser downloads BOQ-[date].pdf
   - Ready for printing or sharing

================================================================================
KNOWN LIMITATIONS
================================================================================

1. Only 2 hierarchy levels (section > subsection > items)
   - Not suitable for deeper nesting
   - Future enhancement possible

2. Item numbering is per-subsection
   - Auto-increment resets for each subsection
   - Manual override possible per item

3. Manual structure creation
   - UI only supports import
   - Structure CRUD requires code/API

4. PDF styling is fixed
   - No custom branding beyond company header
   - Layout is standard across all BOQs

5. No real-time collaboration
   - Single-user editing per instance
   - Multiple users need to refresh

6. No concurrent edit handling
   - Last write wins
   - No merge conflict resolution
